# 글로벌 에스테틱 예약 플랫폼 기획 및 설계서

## 1. 프로젝트 개요 및 비즈니스 전략
* **목적:** 한국을 방문하는 외국인 관광객이 편리하게 뷰티/에스테틱 샵을 예약하고 결제할 수 있는 하이브리드 앱 플랫폼 구축
* **초기 기획 (병원/코인) ➔ 최종 피벗 (에스테틱/카드):**
    * 기존의 '병원 의료 관광' 및 '스테이블코인 결제' 기획은 외국인환자유치업 등록(자본금 1억, 보증보험)과 가상자산 규제(특금법)라는 높은 장벽이 존재함.
    * **전략적 피벗:** 타겟을 '병원'에서 공중위생관리법상 '미용업'인 **'에스테틱'**으로 변경하여 법적/규제적 허들을 완벽히 제거함.
    * **결제 수단:** 복잡한 코인 결제 대신 **해외 신용카드 결제가 가능한 글로벌 PG사(Eximbay, Stripe 등)**를 연동하여 즉시 선결제 또는 가승인/매입(Auth & Capture) 로직을 도입.

## 2. 확정된 기술 스택 (Tech Stack)
* **프론트엔드 (화면):** 순수 HTML / CSS / Vanilla JavaScript
* **클라이언트 (앱):** Android / iOS 네이티브 앱 내 WebView를 활용한 하이브리드 앱
* **백엔드 (서버):** Python Flask (초기 구축의 안정성 및 직관적인 동기식 처리 활용)
* **데이터베이스:** PostgreSQL
* **AI 번역 연동:** PydanticAI + Gemini Flash (`run_sync()` 방식을 사용하여 Flask 환경에서 처리)

## 3. 핵심 기능 명세서 (Feature List)

### 3.1. 사용자용 (외국인 고객 / 하이브리드 앱)
* 이메일 직접 가입 및 소셜 로그인 (Google, Apple, LINE 지원)
* 다국어 UI 기반 에스테틱 샵 검색 및 상세 정보(구글 맵) 확인 — 지원 언어: 영어(en), 일본어(ja), 중국어(zh), 한국어(ko)
* 원하는 날짜/시간 선택, 요청사항 메모 작성 및 PG사 선결제

### 3.2. 상점용 (원장님 / 웹 대시보드)
* 예약 캘린더 확인 및 고객 상태 관리 (노쇼, 방문 완료 처리)
* 스케줄(예약 가능 시간) 열기/닫기 및 정산액 확인
* **고객 요청사항 번역 확인:** 고객의 외국어 요청사항 원문 및 AI 한국어 번역본 동시 노출

### 3.3. 플랫폼 관리자용 (총괄 Admin)
* 입점 샵 관리, 전체 예약 트래픽 모니터링, 수수료 공제 및 정산 처리

### 💡 [핵심] AI 번역 시스템 프롬프트 (System Prompt)
제미나이 Flash API 호출 시, 원장님들의 사용성을 높이고 할루시네이션(환각)을 막기 위해 서버 단에서 고정으로 부여할 프롬프트입니다.
> "너는 한국의 피부관리실 원장님과 외국인 고객 사이의 통역사야. 고객의 말을 한국어로 번역할 때 절대 다른 말(예: '번역해 드리겠습니다' 등)은 덧붙이지 말고 오직 번역된 결과물만 출력해. 전문 피부 미용 용어를 사용해."

* **지원 번역 언어:** 영어(EN), 일본어(JA), 중국어(ZH) 등 모든 외국어 → 한국어 자동 번역

## 4. 데이터베이스 핵심 스키마 설계 (PostgreSQL)

향후 확장성과 결제 무결성을 고려하여 UUID를 기본키로 사용하는 5대 핵심 테이블 구조입니다.

| 테이블명 | 핵심 역할 및 설명 | 주요 컬럼 |
| :--- | :--- | :--- |
| `users` | 통합 가입자 관리 | `id`, `email`, `auth_provider`, `provider_id` |
| `shops` | 에스테틱 상점 정보 | `id`, `name`, `latitude`, `longitude`, `owner_id` |
| `menus` | 상점별 제공 시술 | `id`, `shop_id`, `title`, `price`, `duration` |
| `bookings` | 예약 및 번역 데이터 | `id`, `user_id`, `booking_time`, `status`, `request_original`, **`request_translated`** |
| `payments` | 결제 추적 데이터 | `id`, `booking_id`, `amount`, `pg_tid`, `payment_status` |

## 5. 향후 개발 진행 스텝
1. **로컬 개발 환경 세팅:** Docker 및 `docker-compose.yml`을 활용하여 PostgreSQL 데이터베이스와 Python Flask 서버 컨테이너 구축.
2. **데이터베이스 스키마 생성:** 설계된 5개 테이블의 DDL 작성 및 적용.
3. **API 개발:** 회원가입/로그인 흐름 (JWT 발급) 및 예약 로직 구현.
4. **AI 연동:** PydanticAI를 활용한 예약 메모장 번역 파이프라인 연동.
5. **프론트엔드/웹뷰 연동:** 바닐라 JS 기반 모바일 웹 화면 구현 및 네이티브 앱 패키징.
